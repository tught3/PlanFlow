## 1. 앱 이름 (30자 이내)

```
PlanFlow - 말하면 일정이 되는 앱
```

> 29자. "말하면 일정이 됨" 핵심 가치 + 검색 키워드 "일정"·"앱" 포함.
> 대안 B: `PlanFlow - 음성 일정 관리 알림` (28자, 검색 키워드 강화 버전)

## 2. 짧은 설명 (80자 이내)

```
"내일 2시 강남 미팅" — 말하면 끝. 이동 시간까지 챙겨드리는 일정 앱.
```

> 41자. 후크는 구체적 시나리오로 시작 → 핵심 차별점(이동 시간) → 정체성 한 마디.
> 대안 B: `말 한마디로 일정 등록, 출발 시각까지 알림. 캘린더 연동, 음성 브리핑 제공.` (47자)

## 작성 노트

- **기존 등록 문구** (`docs/play-store-listing.md`)와의 차이: 기존 버전은 기능 나열 중심. 이번 초안은 Pain → 해소 흐름으로 재구성, "생산성/효율" 단어 제거, 사용자 시나리오 선행.
- **캘린더 프레임**: "또 다른 캘린더 앱"이 아니라 "지금 쓰는 캘린더를 더 편하게 쓰는 도구"로 포지셔닝.
- **프라이버시 섹션**: 온디바이스 STT는 경쟁 앱 대비 실질 차별점 → 섹션 독립 유지.
- **CTA**: 현재 베타 전략에 맞춰 "전체 무료 + 얼리버드 신청" 흐름 유지.
- **글자 수**: 전체 설명 약 870자. Google Play 허용 4000자까지 여유 있음. 향후 FAQ 형식 추가 또는 업데이트 내역 섹션 확장 가능.
