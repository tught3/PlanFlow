## 오케스트레이션 흐름

```
요청 수신
  ↓
[계획 단계] claude-opus-4-5
  - 이슈가 2개 이상이거나 여러 서브시스템에 걸친 경우 계획 먼저
  - 수정 방향 확정 후 실행 진입
  ↓
[실행 단계] 워커 서브에이전트 (병렬 가능)
  - 파일 스코프가 겹치지 않으면 병렬 실행
  - 각 워커는 완전히 자립적인 프롬프트(세션 기억 없음)
  ↓
[검토 단계] 리뷰어 에이전트 claude-sonnet-4-5 (필수)
  - 워커 전원 완료 후 실행
  - 계획 대비 실행 결과 검증
  - 문제 발견 시 → 수정 → 재검토 (승인될 때까지 반복)
  ↓
[완료 보고] 리뷰어 승인 + 빌드 통과 후에만 완료 선언
```

---

## 워커 병렬 실행 규칙

- 대상 파일이 겹치지 않으면 동시에 실행
- 각 워커 프롬프트에 포함해야 할 것:
  1. 수정 목적과 배경
  2. 정확한 파일 경로와 변경할 코드 위치
  3. 원하는 결과물의 구체적 명세
  4. "다른 파일은 건드리지 말 것" 범위 제한
- 워커는 결과를 보고 → 오케스트레이터가 통합 → 리뷰어에게 전달

## 리뷰어 체크리스트

리뷰어 에이전트는 다음을 반드시 확인해야 합니다:

- [ ] 변경된 모든 파일이 오류 없이 컴파일되는가
- [ ] 구현 로직이 원래 계획의 의도와 일치하는가
- [ ] 관련 없는 파일이 수정되지 않았는가
- [ ] 알려진 갭이나 잔여 이슈가 명시적으로 보고되었는가
- [ ] 빌드 성공 (`scripts/flutter-local.ps1` 또는 `flutter build apk --debug`)
- [ ] 고위험 영역 수정 시 로직 정확성 이중 확인
