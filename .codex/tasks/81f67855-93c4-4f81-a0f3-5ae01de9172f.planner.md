## 해석된 목표

음성 일정 분석에서 일정 제목·명령어가 장소로 오인되어 자동 좌표화되는 것을 막고, 장소 후보는 근거 점수에 따라 자동 확정·확인 요청·미설정으로 구분한다.

## 요구사항 / 비요구사항

- 명사만인 후보는 자동 장소 확정 금지.
- `중요 일정`, 반복·저장·알람 명령, 날짜·시간을 포함한 장소 후보는 거절한다. 단, 독립적으로 추출된 명시적 장소 span이 있으면 그것은 유지한다.
- 장소 후보 점수:
  - 조사 또는 장소 유형 접미사, 이동 맥락: 긍정 근거
  - 지도 결과의 정규화 유사도 `>= 0.51`, 히스토리 일치: 추가 근거
- 점수별 처리:
  - `>= 0.75`: 자동 좌표 해석
  - `0.45–0.74`: 좌표 자동 확정 없이 순위화된 확인 후보
  - `< 0.45`: 장소 비움, 제목 보존
- DB 마이그레이션, 스키마 변경, 기존 일정 제목 의미 변경은 범위 밖.

## 영향 및 위험

- 핵심 위험: 거절 규칙이 넓으면 정상 장소를 누락할 수 있음. 따라서 “명령어 포함”만으로 제거하지 않고 독립적으로 추출된 명시적 장소 span을 우선 보존해야 한다.
- 점수 기준은 단일 noun 보정이 아니라 여러 근거의 합산이어야 하며, 지도 유사도만으로 `.75`에 도달하지 않도록 설계한다.
- 확인 구간은 좌표를 확정하거나 저장하지 않아야 한다. 후보 순위만 노출 가능한 구조화 결과로 남긴다.
- 공개 cutoff API는 기존 private relevance 계산과 의미가 충돌하지 않도록 기존 정렬 동작을 유지한다.

## 검증

구현자는 프로젝트의 기존 테스트 명령을 우선 사용하며, 최소 증거는 아래와 같다.

```powershell
flutter test test/**/voice_command_analysis_service*_test.dart
flutter test test/**/voice_schedule_structure_service*_test.dart
flutter test test/**/location_lookup_service*_test.dart
flutter analyze
```

기대 증거:

- `태블릿계기판 중요한일정` → 장소/좌표 없음, 제목 유지
- `강남역에서 중요한 일정`처럼 독립 장소 span이 있는 입력 → `강남역`만 장소 후보로 유지
- 지도 유사도 `< 0.51`은 그 근거를 획득하지 않음
- 총점 `.75` 이상만 자동 좌표화
- `.45–.74`는 정렬된 확인 후보, 자동 좌표화 없음
- `.45` 미만은 빈 장소
- 기존 명시적 정상 장소 입력의 회귀 테스트 통과

## 병렬화
