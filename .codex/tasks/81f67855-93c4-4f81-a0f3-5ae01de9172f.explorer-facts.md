**Goal**
- Prevent model-provided `schedule_fields.location` from accepting command-like text such as `태블릿계기판 중요한일정` as a real location.
- Preserve existing behavior for valid explicit places and current time-only rejection.

**Root Cause**
- `VoiceCommandAnalysisService._normalizeScheduleFields` forwards `schedule_fields.location` into `VoiceScheduleStructureService.normalizeScheduleLocation`.
- `normalizeScheduleLocation` accepts a non-empty explicit candidate after `_normalizeLocationCandidate`.
- `_isInvalidLocationCandidate` only rejects date/time metadata and a few verbal fragments, so it does not block `태블릿계기판 중요한일정` or strings containing the importance command.
- `LocationLookupService` can rank relevance, but there is no public normalized similarity threshold that auto-rejects ambiguous command-like text before coordinate resolution.

**Affected Symbols**
- `VoiceCommandAnalysisService._normalizeScheduleFields`
- `VoiceScheduleStructureService.normalizeScheduleLocation`
- `VoiceScheduleStructureService._normalizeLocationCandidate`
- `VoiceScheduleStructureService._isInvalidLocationCandidate`
- `LocationLookupService` relevance sorting / coordinate-resolution path

**Minimal Implementation Surface**
- Tighten `VoiceScheduleStructureService._isInvalidLocationCandidate` to reject command-bearing or command-adjacent strings, including the reported phrase.
- Keep `VoiceCommandAnalysisService._normalizeScheduleFields` unchanged unless tests show the caller needs a guard before normalization.
- Leave `LocationLookupService` unchanged unless the fix needs a second-stage ambiguity cutoff; current facts only show sorting, not a public threshold.

**Focused Regression Tests**
- Reject `schedule_fields.location = "태블릿계기판 중요한일정"` during schedule-field normalization.
- Reject location strings that contain the importance command fragment, even when prefixed by a plausible place token.
- Continue rejecting time-only / date-only location candidates, preserving existing coverage.
- Continue accepting explicit place extraction cases already covered by current tests.
- Add one negative test that asserts no coordinate lookup or normalized location output is produced for the reported phrase.
