## 2. 스마트 준비 알람 개선

### 2-1. DB 변경
```sql
alter table user_settings
  add column prep_time_min integer default 30,
  add column prep_pre_alarm_offset integer default 30,
  add column depart_pre_alarm_offset integer default 30;
-- prep_time_min: 나의 평균 준비 시간 (분)
-- prep_pre_alarm_offset: 준비 시작 사전 예고 알림 (분 전, 0=안받기)
-- depart_pre_alarm_offset: 출발 사전 예고 알림 (분 전, 0=안받기)
```

### 2-2. 알람 4단계 구조
모든 외부 일정에 아래 순서로 알람 생성:
```
출발 시각 = start_at - 이동시간 - 여유 30분

[첫 외부 일정만]
① 준비 사전 예고 = 출발시각 - prep_time_min - prep_pre_alarm_offset
   메시지: "N분 뒤부터 준비 시작하세요 🔔"
   → prep_pre_alarm_offset == 0이면 생략

② 준비 시작 알람 = 출발시각 - prep_time_min
   메시지: "지금 준비 시작하세요 🚿"

[모든 외부 일정]
③ 출발 사전 예고 = 출발시각 - depart_pre_alarm_offset
   메시지: "N분 뒤 출발해야 해요 🔔"
   → depart_pre_alarm_offset == 0이면 생략

④ 출발 알람 = 출발시각
   메시지: "지금 출발하세요 🚗 (이동 약 N분)"
```

예시 (12시 원주, 이동 90분, 준비 60분, 사전알림 30분):
```
08:30 → "30분 뒤부터 준비 시작하세요 🔔"
09:00 → "지금 준비 시작하세요 🚿"
09:30 → "30분 뒤 출발해야 해요 🔔"
10:00 → "지금 출발하세요 🚗 (이동 약 90분)"
```

### 2-3. 중복/음수 방지
- 각 알람 시각이 현재 시각보다 과거면 생략
- 알람 간격이 5분 미만으로 겹치면 하나로 합침
- prep_pre_alarm_offset == 0이면 ① 생략
- depart_pre_alarm_offset == 0이면 ③ 생략

## 4. 외부 일정 판단 로직

아래 두 조건 모두 충족 시에만 준비/출발 알람 생성:
- 조건 1: location 필드가 비어있지 않음
- 조건 2: location 또는 title에 내부/집 키워드 없음

```dart
const internalKeywords = [
  '집', '재택', '자택', '집에서', '집앞',
  '온라인', '화상', '줌', 'zoom', 'zep', 'webex',
  '구글밋', 'google meet', 'teams', 'ms teams',
  '전화', '통화', '콜', '전화회의', '컨퍼런스콜',
  '내부', '사내', '팀내', '오피스', '자체',
];

bool isExternalEvent(EventModel event) {
  if (event.location == null || event.location!.trim().isEmpty) return false;
  final location = event.location!.toLowerCase();
  final title = event.title.toLowerCase();
  for (final keyword in internalKeywords) {
    if (location.contains(keyword) || title.contains(keyword)) return false;
  }
  return true;
}
```

첫 외부 일정 판단:
```dart
// 당일 외부 장소 있는 일정 중 start_at이 가장 이른 일정 = 첫 외부 일정
Future<bool> isFirstExternalEventOfDay({
  required String userId,
  required DateTime date,
  required String eventId,
}) async {
  final dayEvents = await eventRepository.fetchByDate(userId: userId, date: date);
  final externalEvents = dayEvents
      .where((e) => isExternalEvent(e))
      .toList()
    ..sort((a, b) => a.startAt.compareTo(b.startAt));
  return externalEvents.isNotEmpty && externalEvents.first.id == eventId;
}
```

## 5. 병원 등 장소 맥락 판단 개선

### 현재 문제
"병원" 키워드만 감지하면 무조건 진료 목적으로 판단해서
금식/준비 알람을 생성함.

### 수정
GPT 파싱 프롬프트에서 선행행동 생성 시
장소 키워드만 보지 말고 행동 동사 + 장소 조합으로 판단:

| 키워드 | 판단 | 선행행동 |
|--------|------|---------|
| "진료", "검사", "수술", "입원", "시술" | 의료 목적 | 금식/보험카드 알람 |
| "미팅", "영업", "방문", "상담", "계약" | 업무 목적 | 일반 이동 알람만 |
| "병문안", "문병" | 방문 목적 | 꽃/선물 챙기기 알람 |
| 동사 없이 장소만 | 불분명 | ConfirmScreen에서 목적 선택 UI |

불분명 시 ConfirmScreen 목적 선택 UI:
```
🏥 병원 일정이 감지됐어요
방문 목적이 무엇인가요?
[진료/검사]  [업무/영업]
[병문안]     [기타]
```

병원 외 동일 원칙 적용:
- "법원" → 소송/재판 vs 업무 방문
- "학교" → 학부모 상담 vs 업무
장소만으로 목적을 단정하지 말 것.
GPT 프롬프트 few-shot 예시도 이에 맞게 업데이트.
