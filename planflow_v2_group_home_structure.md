# PlanFlow V2 팀 홈/그룹 운영 화면 구성안

## 목적

PlanFlow V2의 그룹 기능은 단순히 버튼을 나열하는 구조가 아니라, 사용자가 특정 팀/그룹에 들어갔을 때 바로 “팀 운영 홈”처럼 이해할 수 있어야 한다.

현재 이미 구현된 기능은 다음과 같다.

- 그룹 생성
- 초대 관리
- 멤버 관리
- 그룹 일정
- 대시보드
- 캘린더 오버레이

다음 단계에서는 이 기능들을 팀 홈 기준으로 정리하고, 부족한 설정/권한/하위그룹 UX를 후속 단계로 분리한다.

---

## 1. 팀 홈 기본 구조

팀에 들어가면 상단에 현재 팀 정보를 먼저 보여준다.

예시:

```text
서울1팀
내 역할: 리더
멤버 5명
오늘 일정 3개
이번 주 일정 12개
```

필수 표시:

- 팀명
- 내 역할: leader/member
- 멤버 수
- 오늘 일정 수
- 이번 주 일정 수
- 현재 선택된 팀 여부
- 개인 모드로 돌아가기
- 팀 전환 진입점
- 팀 설정 진입점

---

## 2. 팀 홈 메뉴 구조

모바일에서는 탭을 너무 많이 두면 복잡하므로, 기본 화면은 “요약 카드 + 빠른 버튼 + 더보기” 구조가 적절하다.

추천 UI:

```text
[팀 요약 카드]

서울1팀
리더
멤버 5명
오늘 일정 3개
이번 주 일정 12개

[그룹 일정] [멤버 관리]
[초대 관리] [대시보드]

더보기
- 팀 설정
- 권한 위임
- 하위 그룹
```

## 9. 하위 그룹 UX

PlanFlow V2의 조직 구조는 고정된 본부장/팀장/팀원이 아니라 Group Tree 구조다.

즉 다음 구조를 지원해야 한다.

```text
A 본부장
 ├─ B 1팀 팀장
 │   ├─ D 1팀 팀원
 │   └─ E 1팀 팀원
 └─ C 2팀 팀장
     ├─ G 2팀 팀원
     └─ T 2팀 팀원
```

B는 A 입장에서는 멤버지만, D/E 입장에서는 리더다.

### V2에서 필요한 최소 UX

- 현재 그룹의 하위 그룹 표시
- 하위 그룹 생성 진입점
- 하위 그룹으로 이동
- 상위 그룹으로 이동

### V2.1로 미뤄도 되는 것

- 조직도 시각화
- 드래그 앤 드롭 조직 이동
- 대량 멤버 이동
- 하위 그룹 통계 상세화

## 11. 현재 구현 기준 부족한 점

현재 이미 있는 것:

- 대시보드
- 그룹 일정
- 초대 관리
- 멤버 관리
- 캘린더 오버레이

부족한 것:

- 팀 홈 정보 구조가 약함
- 팀 설정이 없음
- 권한 위임 UI가 없음
- 멤버 이름 표시가 약함
- 그룹 일정 수정 화면이 약함
- 하위 그룹 생성 UX가 약함
- 팀 전환 UX가 더 명확해야 함

## 12. 다음 구현 추천

다음 구현은 새 대형 기능보다 “팀 홈 정리”가 우선이다.

```text

```

그룹 관리 화면을 단순 버튼 목록이 아니라 팀 홈 구조로 정리한다.

- 현재 팀명/역할/멤버 수/오늘 일정/이번 주 일정 표시
- 빠른 버튼 4개 정리
  - 그룹 일정
  - 멤버 관리
  - 초대 관리
  - 대시보드
- 더보기 섹션 추가
  - 팀 설정
  - 권한 위임
  - 하위 그룹
- 개인 모드/팀 전환 표시 개선

### 금지

- 회의록 구현 금지
- Action Item 구현 금지

- AI 코칭 구현 금지

## 결론

PlanFlow V2의 다음 UX 목표는 “기능 추가”가 아니라 “팀에 들어갔을 때 팀 운영 홈처럼 보이게 하는 것”이다.

따라서 다음 작업은 다음 순서가 적절하다.

1. Group Home UX 정리
2. 팀 설정 최소 화면
3. 권한 위임 UI
4. 하위 그룹 생성 UX
5. 그룹 일정 수정 화면
6. 멤버 프로필 이름 표시
