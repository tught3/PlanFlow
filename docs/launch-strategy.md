## ORB 채널 전략

### Owned (소유 채널)

**1순위: 얼리버드 이메일 리스트**
- 앱 내 얼리버드 신청 기능 (early_bird_emails 테이블)
- GitHub Pages 랜딩페이지 이메일 수집 폼
- 2차 유료화 시 쿠폰 발송의 핵심 자산

**2순위: GitHub Pages 랜딩페이지**
추가 필요 항목:
- "말 한마디로 일정 등록" 헤드라인
- 30초 데모 GIF (음성 입력 → 일정 생성 흐름)
- 얼리버드 신청 CTA
- 프라이버시 뱃지 ("음성은 내 기기 안에서만")

---

### Rented (임차 채널) — 한국 시장 특화

| 채널 | 타깃 | 콘텐츠 |
|------|------|--------|
| 블라인드 | 직장인, 영업직 | "이런 앱 없나 했는데..." 실사용 후기형 |
| 네이버 카페 | 영업직/현장직 커뮤니티 | 기능 소개 + 베타 초대 |
| 디스콰이엇 (disquiet.io) | 한국판 ProductHunt | 제품 등록 + 메이커 스토리 |
| 인스타그램 | 비즈니스 직장인 | 기능 데모 릴스 (30초) |
| 링크드인 | 영업/비즈니스 전문직 | "지각 없는 하루" 스토리형 포스트 |

핵심 원칙: 모든 채널 → 랜딩페이지 이메일 수집으로 유도

---

### Borrowed (차용 채널)

1. 네이버 블로그/유튜브 리뷰어에게 베타 초대권 무료 제공
   - "생산성 앱 리뷰" 유튜버 (구독자 1만~10만 구간)
   - 영업/현장직 직장인 인플루언서
2. 블라인드 개발자 인증 후 직접 출시 게시글
3. 네이버 카페 "외근/영업직" 카테고리 직접 게시
4. K-Startup 한양대학교 파트너십 네트워크 활용

## Pre-Launch 체크리스트

### 🔴 배포 전 필수

- [x] Play Store 등록정보 ASO 개선 완료
- [x] 개인정보처리방침 URL
- [x] 계정 삭제 URL
- [x] 스크린샷 5장 준비
- [ ] 랜딩페이지 이메일 수집 폼 작동 확인 (GitHub Pages)
- [ ] 앱 내 얼리버드 신청 → early_bird_emails 저장 확인
- [ ] Play Store 스크린샷 업로드 완료

### 🟡 배포 당일

- [ ] 디스콰이엇(disquiet.io) 제품 등록
- [ ] 블라인드 "앱 출시" 게시글 (직장인 카테고리)
- [ ] 네이버 카페 1-2곳 게시
- [ ] 인스타그램/링크드인 출시 포스트

### 🟢 배포 후 1주일 내

- [ ] 초기 사용자 피드백 수집 (앱 내 피드백 기능)
- [ ] 얼리버드 이메일 수신자에게 출시 알림 발송
- [ ] 리뷰어 1-2명에게 베타 초대권 발송

## 얼리버드 이메일 시퀀스

```
[즉시] 신청 확인
"PRO 얼리버드 신청 완료! 지금 PlanFlow를 무료로 쓰세요."
→ Play Store 다운로드 링크

[2차 배포 D-7] 예고
"곧 PRO가 출시됩니다. 얼리버드 할인 쿠폰을 준비했어요."

[2차 배포 D-day] 쿠폰 발송
"얼리버드 전용 30% 할인 쿠폰입니다."
```

## 핵심 메시지 (전 채널 공통)

> **"말하면 일정이 됩니다. 출발 시간까지."**

- Before: "10시 미팅인데 몇 시에 나가야 하지? 강남까지 30분... 아 늦겠다"
- After: "9시 20분에 출발하세요" 알림 + 음성으로 다음 일정 바로 등록
