# PlanFlow V2 Product Structure

## 1. V2 제품 한 줄 정의

PlanFlow V2는 개인 AI 스케줄러를 유지하면서, 팀장이 팀 일정과 회의 후속조치를 관리할 수 있게 해주는 AI 팀 운영 비서다.

## 2. V2 핵심 원칙

- 개인 일정은 기존 PlanFlow 구조를 유지한다.
- 팀 일정은 개인 일정과 섞지 않고 별도 `team_events`로 관리한다.
- 개인 일정과 팀 일정은 캘린더 화면에서 오버레이로 함께 보여준다.
- 팀 공유 일정은 팀원 전체가 볼 수 있다.
- 개인 일정은 팀에 자동 공유하지 않는다.
- 회의록과 액션아이템은 분리한다.
- AI가 만든 액션아이템은 바로 Task가 아니라 후보 상태로 저장한다.
- 팀장 확인 후 Task로 승격한다.
- V2에서는 복잡한 프로젝트 관리, 칸반, AI 코칭, 성과 분석은 제외한다.

## 3. 사용자 역할 구조

### 본부장/관리자

#### 역할 설명

- 여러 팀을 한 번에 보는 상위 관리 역할이다.
- V2에서는 정교한 조직도 관리자라기보다, 팀들을 묶어 읽는 상위 관찰자에 가깝다.

#### V2 MVP에 포함되는 범위

- 팀 목록 조회
- 팀별 공유 일정 조회
- 팀장별 진행 상황 조회
- 읽기 중심 확인

#### 나중에 확장할 범위

- 여러 팀 간 비교
- 조직 단위 브리핑
- 본부 관점의 통합 리포트

#### 볼 수 있는 정보

- 팀 목록
- 팀 일정
- 팀별 회의록 요약
- 팀별 액션아이템 진행 상태

#### 수정할 수 있는 정보

- V2 MVP에서는 제한적이거나 없음
- 이후 확장 시 팀 운영 정책 범위로 일부 허용 가능

### 팀장

#### 역할 설명

- 팀의 일정과 회의 후속조치를 실제로 관리하는 핵심 역할이다.
- V2에서 가장 중요한 사용자다.

#### V2 MVP에 포함되는 범위

- 팀 생성
- 팀원 초대
- 팀 일정 생성/수정/삭제
- 회의록 입력
- AI 요약 결과 검토
- 액션아이템 승인
- Task 생성

#### 나중에 확장할 범위

- 프로젝트 관리
- 팀 주간 브리핑
- 팀 성과 확인
- 반복 업무 관리

#### 볼 수 있는 정보

- 팀 전체 일정
- 팀원별 할당 Task
- 회의록 요약
- 액션아이템 후보

#### 수정할 수 있는 정보

- 팀 정보
- 팀 일정
- 회의록
- 액션아이템 승인/거절
- Task 담당자와 마감일

### 팀원

#### 역할 설명

- 팀 일정과 본인 Task를 확인하고 실행하는 역할이다.

#### V2 MVP에 포함되는 범위

- 팀 공유 일정 확인
- 본인 Task 확인
- 회의 참석
- 본인 할당 업무 상태 변경

#### 나중에 확장할 범위

- 회의록 코멘트
- Task 세부 코멘트
- 일정 제안
- 업무 제안

#### 볼 수 있는 정보

- 팀 공유 일정
- 본인에게 할당된 Task
- 회의 결과 요약

#### 수정할 수 있는 정보

- 본인 Task 상태
- 본인 코멘트
- 일부 회의 메모 입력

## 4. 사용자별 하루 사용 시나리오

### 팀장 시나리오

아침 앱 실행 → 오늘 팀 일정 확인 → 미완료 액션아이템 확인 → 팀 일정 생성 → 회의 진행 → 회의록 입력 → AI 요약 → 액션아이템 확인 → 담당자/마감일 지정 → Task 생성 → 팀원 진행상태 확인

### 팀원 시나리오

아침 앱 실행 → 개인 일정 확인 → 팀 공유 일정 확인 → 자신에게 할당된 Task 확인 → 회의 참석 → 할당된 업무 상태 변경 → 완료 처리

### 본부장/관리자 시나리오

앱 실행 → 여러 팀 목록 확인 → 팀별 공유 일정 확인 → 팀장별 진행 상황 확인

단, V2 MVP에서는 읽기 중심 또는 후순위로 둔다.

## 5. 핵심 파이프라인

### 개인 일정 파이프라인

개인 음성/수동 입력 → `events` 저장 → 개인 캘린더 표시 → 개인 알림

### 팀 일정 파이프라인

팀장 또는 권한 있는 팀원 → 팀 일정 생성 → `team_events` 저장 → 팀원 전체 조회 → 캘린더에 팀 일정 오버레이 표시

### 회의록 파이프라인

팀 일정 또는 회의록 메뉴 → 회의록 입력 → `meeting_notes` 저장 → AI 요약 → 결정사항 저장 → 액션아이템 후보 생성

### 액션아이템 파이프라인

AI 후보 생성 → 팀장 검토 → 담당자 지정 → 마감일 지정 → 승인 → `tasks` 생성

### Task 파이프라인

Task 생성 → 담당자에게 표시 → 진행중 → 완료 → 팀장 확인

## 6. 화면 구조 초안

### 개인 홈

- 화면 목적: 개인 일정과 오늘의 우선순위를 보여준다.
- 주요 버튼: 음성 입력, 일정 추가, 캘린더 이동
- 표시 데이터: 개인 일정, 알림, 브리핑
- 진입 경로: 앱 실행 후 기본 진입
- V2 MVP 포함 여부: 포함

### 캘린더

- 화면 목적: 개인 일정과 팀 일정을 한 화면에서 구분해 보여준다.
- 주요 버튼: 일정 생성, 필터, 보기 전환
- 표시 데이터: `events`, `team_events`
- 진입 경로: 개인 홈, 팀 홈
- V2 MVP 포함 여부: 포함

### 팀 홈

- 화면 목적: 팀의 핵심 상태를 모아 보여준다.
- 주요 버튼: 팀 생성, 초대, 팀 캘린더, 회의록 작성
- 표시 데이터: 팀 정보, 팀 일정, 미완료 액션아이템
- 진입 경로: 앱 내 팀 탭
- V2 MVP 포함 여부: 포함

### 팀 생성

- 화면 목적: 새 팀을 만든다.
- 주요 버튼: 생성, 취소
- 표시 데이터: 팀명, 설명, 기본 설정
- 진입 경로: 팀 홈, 초대 흐름
- V2 MVP 포함 여부: 포함

### 팀원 초대

- 화면 목적: 팀원을 초대하고 초대 상태를 관리한다.
- 주요 버튼: 초대 생성, 초대 링크 복사, 재전송
- 표시 데이터: 초대 목록, 만료 상태
- 진입 경로: 팀 홈, 팀원 목록
- V2 MVP 포함 여부: 포함

### 팀원 목록

- 화면 목적: 팀 구성과 역할을 확인한다.
- 주요 버튼: 역할 확인, 초대 관리, 멤버 상세
- 표시 데이터: 팀원, 역할, 초대 상태
- 진입 경로: 팀 홈, 팀원 초대
- V2 MVP 포함 여부: 포함

### 팀 캘린더

- 화면 목적: 팀 일정을 확인하고 관리한다.
- 주요 버튼: 팀 일정 추가, 상세 보기, 수정, 삭제
- 표시 데이터: `team_events`
- 진입 경로: 팀 홈, 캘린더
- V2 MVP 포함 여부: 포함

### 팀 일정 생성/상세/수정

- 화면 목적: 팀 일정을 등록하고 관리한다.
- 주요 버튼: 저장, 수정, 삭제
- 표시 데이터: 제목, 시간, 장소, 참석자, 메모
- 진입 경로: 팀 캘린더, 팀 홈
- V2 MVP 포함 여부: 포함

### 회의록 작성

- 화면 목적: 회의 내용을 입력한다.
- 주요 버튼: 저장, AI 요약 요청
- 표시 데이터: 회의 텍스트, 참석자, 메모
- 진입 경로: 팀 홈, 팀 일정 상세
- V2 MVP 포함 여부: 포함

### AI 회의 요약 결과

- 화면 목적: 회의 요약, 결정사항, 액션아이템 후보를 보여준다.
- 주요 버튼: 재요약, 액션아이템 확인, 저장
- 표시 데이터: 요약, 결정사항, 후보 액션아이템
- 진입 경로: 회의록 작성 후
- V2 MVP 포함 여부: 포함

### 액션아이템 확인

- 화면 목적: AI가 만든 후보를 팀장이 검토하게 한다.
- 주요 버튼: 승인, 거절, 담당자 지정, 마감일 지정
- 표시 데이터: 액션아이템 후보, 신뢰도, 추천 담당자
- 진입 경로: AI 회의 요약 결과
- V2 MVP 포함 여부: 포함

### 내 Task 목록

- 화면 목적: 팀원이 본인 할 일을 확인한다.
- 주요 버튼: 상태 변경, 상세 보기
- 표시 데이터: 본인 할당 Task
- 진입 경로: 팀 홈, 알림
- V2 MVP 포함 여부: 포함

### Task 상세

- 화면 목적: Task의 상세 내용과 진행 상태를 보여준다.
- 주요 버튼: 상태 변경, 코멘트, 완료 처리
- 표시 데이터: 담당자, 마감일, 진행 상태, 관련 회의 맥락
- 진입 경로: 내 Task 목록, 액션아이템 승인 후
- V2 MVP 포함 여부: 포함

## 7. UX 원칙

- 개인 일정과 팀 일정은 색상/라벨로 확실히 구분한다.
- 팀 일정은 `팀` 배지를 붙인다.
- 개인 일정은 절대 자동 공유하지 않는다.
- AI가 만든 액션아이템은 사용자가 확인하기 전까지 확정 업무로 보이지 않게 한다.
- 팀장은 `검토 후 확정` 흐름을 가져야 한다.
- 팀원은 복잡한 관리 화면보다 `내가 해야 할 일` 중심으로 본다.
- 본부장/관리자 기능은 V2에서는 과하게 만들지 않는다.

## 8. V2에서 절대 하지 않을 것

- 개인 `events` 테이블에 `team_id`를 억지로 붙이는 설계
- 개인 일정을 자동으로 팀에 공유하는 설계
- AI 액션아이템을 검토 없이 바로 Task로 만드는 설계
- 프로젝트 관리 전체 기능
- 칸반 보드
- 성과 분석
- AI 팀 코칭
- 외부 캘린더 양방향 팀 동기화
- 복잡한 본부/회사 조직도 관리

## 9. 최종 결론

V2의 핵심은 협업툴 전체가 아니라, “팀장이 팀 일정과 회의 후속조치를 놓치지 않게 해주는 AI 팀 비서”다.
