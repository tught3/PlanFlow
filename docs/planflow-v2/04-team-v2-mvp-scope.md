# PlanFlow Team V2 MVP Scope

## 1. 2차 MVP의 목적

2차 MVP는 완성형 협업툴이 아니라, 개인 AI 스케줄러인 PlanFlow가 팀 단위에서도 작동할 수 있는지 검증하는 실험 버전이다.

핵심 검증 질문은 다음과 같다.

- 사용자가 팀을 만들고 사람을 초대할 수 있는가?
- 팀원이 같은 팀 일정을 함께 볼 수 있는가?
- 팀 일정을 생성/수정할 수 있는가?
- 회의 내용을 AI로 요약할 수 있는가?
- 회의에서 실행해야 할 일을 자동 추출할 수 있는가?
- 개인 일정 구조를 망가뜨리지 않고 팀 기능을 추가할 수 있는가?

## 2. 2차 MVP 포함 범위

### A. 팀 기반

반드시 포함한다.

- 팀 생성
- 팀 목록
- 팀 상세
- 팀원 초대
- 초대 수락
- 팀원 권한 표시

### B. 팀 캘린더

반드시 포함한다.

- 팀 캘린더 조회
- 팀 일정 생성
- 팀 일정 상세
- 팀 일정 수정
- 팀 일정 삭제
- 개인 일정과 팀 일정 시각적 구분

### C. 회의록 AI

반드시 포함한다.

- 회의록 생성
- 회의 내용 텍스트 입력
- AI 회의 요약
- 결정사항 추출
- 액션아이템 추출
- 액션아이템 담당자/마감일 후보 제안

### D. 최소 권한 정책

반드시 포함한다.

- owner
- admin
- member
- viewer

단, MVP에서는 owner/member 중심으로 단순화해도 된다.

## 3. 2차 MVP 제외 범위

이번 MVP에서 제외한다.

- 완성형 프로젝트 관리
- 칸반 보드
- 복잡한 태스크 상태 관리
- 반복 업무 자동화
- AI 팀 코칭 리포트
- 팀 성과 분석
- 개인 일정 일부 공유
- 외부 캘린더와 팀 캘린더 양방향 동기화
- Slack/Notion/Google Workspace 연동
- 결제/요금제
- 관리자 웹 대시보드

### 제외 이유

- 범위가 커지면 1차 배포 안정화에 영향이 생긴다.
- 개인 MVP가 오염될 위험이 있다.
- 팀 기능 검증 전에는 과도한 구현이 된다.
- AI 코칭은 충분한 팀 데이터가 쌓인 뒤에 의미가 있다.

## 4. 2차 MVP 핵심 사용자 흐름

### Flow 1. 팀 생성

사용자 → 팀 만들기 → 팀명 입력 → 팀 생성 완료 → 팀 홈 진입

### Flow 2. 팀원 초대

팀 owner → 초대 버튼 → 이메일 입력 → 초대 생성 → 상대가 수락 → 팀원 목록 반영

### Flow 3. 팀 일정 생성

팀 홈 또는 팀 캘린더 → 일정 추가 → 제목/시간/장소/참석자 입력 → 저장 → 팀원들이 일정 확인

### Flow 4. 회의록 요약

팀 회의록 화면 → 회의 텍스트 입력 → AI 요약 요청 → 요약/결정사항/액션아이템 생성

### Flow 5. 액션아이템 생성

AI가 추출한 액션아이템 → 담당자/마감일 확인 → 저장 → 팀 업무 목록에 표시

## 5. 2차 MVP 성공 기준

### 정량 기준

- 팀 생성 성공
- 초대 수락 성공
- 팀 일정 CRUD 성공
- 회의록 요약 성공
- 액션아이템 1개 이상 자동 추출 성공
- 개인 일정 기능 회귀 없음

### 정성 기준

- 사용자가 “팀 일정 공유 앱”이 아니라 “AI가 회의 후 할 일을 정리해주는 팀 비서”라고 느껴야 한다.
- 개인 일정과 팀 일정이 헷갈리지 않아야 한다.
- 팀 기능이 기존 개인 기능을 방해하지 않아야 한다.

## 6. 구현 우선순위

### P0

- `teams`
- `team_members`
- `team_invites`
- `team_events`
- 팀 생성/초대/조회
- 팀 캘린더
- 팀 일정 CRUD

### P1

- `meeting_notes`
- `meeting_action_items`
- 회의 요약
- 액션아이템 추출
- 액션아이템 저장

### P2

- `projects`
- `tasks`
- 간단 업무 목록
- 담당자/마감일 관리

### 제외

- `coaching_reports`
- 고급 프로젝트 관리
- 통계/성과 분석

## 7. 2차 이후 확장 후보

MVP 검증 후 추가할 수 있다.

- AI 팀 코칭 리포트
- 프로젝트별 일정/업무 묶음
- 팀 주간 브리핑
- 업무 병목 분석
- 개인 일정 일부 공유
- 팀 알림
- 외부 협업툴 연동

## 8. 최종 결론

PlanFlow Team V2의 MVP는 협업툴 전체를 만드는 것이 아니라, “AI 기반 팀 일정 + 회의 후속조치 정리”가 실제로 사용 가치가 있는지 검증하는 버전이다.

따라서 2차 MVP는 팀 생성, 팀 캘린더, 팀 일정, 회의록 요약, 액션아이템 추출까지만 포함한다.
