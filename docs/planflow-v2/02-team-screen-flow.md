# PlanFlow 2차 팀 기능 화면 흐름 초안

이 문서는 PlanFlow 2차 팀 기능의 화면 구성을 정리한 초안이다.

## 1. 공통 원칙

- 개인 MVP 화면과 팀 화면은 기능상 분리한다.
- 팀 기능은 별도 진입 경로를 가진다.
- 화면마다 데이터 소스와 권한 조건을 분명히 둔다.
- 1차 배포에 없는 기능은 MVP 제외로 명시한다.

## 2. 팀 홈

### 목적

- 팀 전체 현황을 한눈에 보여준다.
- 최근 일정, 프로젝트, 태스크, 회의록, 코칭 리포트로 진입하는 허브 역할을 한다.

### 주요 UI 요소

- 팀 이름과 요약 정보
- 오늘/이번 주 팀 일정 카드
- 진행 중 프로젝트 카드
- 나의 할 일 카드
- 최근 회의록 카드
- AI 코칭 요약 카드
- 팀 메뉴 바로가기

### 진입 경로

- 앱 내 팀 탭
- 팀 캘린더/프로젝트/태스크/회의록/리포트 화면에서 뒤로가기 후 복귀

### 연결되는 데이터

- `teams`
- `team_events`
- `projects`
- `tasks`
- `meeting_notes`
- `coaching_reports`

### MVP 포함 여부

- 포함

### 나중에 해도 되는 요소

- 개인화된 팀 인사이트 추천
- 팀 간 비교 대시보드
- 고급 필터

## 15. 화면 흐름 요약

권장 흐름은 다음과 같다.

1. 팀 홈에서 진입한다.
2. 팀 초대 또는 팀 생성으로 팀을 만든다.
3. 팀원 관리로 멤버 상태를 정리한다.
4. 팀 캘린더에서 팀 일정을 확인한다.
5. 프로젝트와 태스크로 작업을 관리한다.
6. 회의록에서 액션아이템을 추출한다.
7. AI 팀 코칭 리포트로 운영 패턴을 점검한다.

이 흐름은 팀 기능을 하나의 업무 루프로 묶기 위한 기본 경로다.
