## 2. 사실관계 검증

### 2-1. 4번 — 캘린더 동기화 A안 vs B안 비교
- **A안**("그대로 가져오세요"/"확인하세요"): "가져오기·조회"만 암시 → 실제 구현(양방향 자동 동기화, PUT export + 일일 pull)을 **축소** 전달.
- **B안**("한곳에서"/"자동으로 동기화하세요"): 이점(한곳에서) 우선 + 정확한 기능(자동 동기화) 유지.
- **판정: B안 채택.** 근거는 기존 v2 검증(NaverCalDavService.exportEvent PUT, DailyCalendarSyncSchedulerService, 앱 UI "자동 동기화 중" 라벨) 그대로 유효.

### 2-2. 7번 — "역할·댓글" 문구 검증
코드 확인: `lib/features/groups/repositories/group_event_comment_repository.dart`(댓글 기능 실존), `group_member_model.dart`/`group_delegation_repository.dart` 등 다수 파일에 역할(리더/멤버) 기반 로직 확인.
**판정: "초대·역할·댓글" 문구 실제 기능과 일치, 과장 아님.**

### 2-3. 6번 — 화면·문구 일치 재확인
v2에서 이미 "브리핑 설정 화면 ≠ 결과 화면" 문제를 해결(옵션 B 적용)했으며 이번 라운드도 동일 화면 재사용. **일치 확인.** 메모: *실제 모닝/이브닝 브리핑 결과 화면 캡처가 확보되면 6번을 결과 화면 기반으로 교체 검토.*

### 2-4. 8번 — CTA 과도한 광고 여부
"지금 시작하세요" — 가격/할인/한정 등 판매 압박 어휘 없음, 앱 내 실제 진입 유도 수준. 무료 정책 미확정 전제로 "무료" 표현 배제. **과도한 광고 아님.**

## 3. Feature Graphic A~D 비교 및 순위

| 기준 | A 슬로건 | B 흐름 | C 차별점 | D 실UI+슬로건 |
|---|---|---|---|---|
| 첫눈에 이해 | 빠름 | 보통(4단계 읽어야 함) | 빠름 | 빠름 |
| 차별점 노출 | 약함 | 있음(전체 흐름) | 강함(출발알림) | 강함(출발알림 배지+실UI) |
| 제품 신뢰 | 낮음(추상 아이콘) | 낮음 | 낮음 | **높음(실제 화면 증거)** |
| 스크린샷과 중복 | 높음(1·8번과 텍스트 동일) | 낮음 | 보통 | 보통(헤드라인은 겹치나 실UI로 차별화) |
| 축소 가독성 | 좋음 | 위험(pill 텍스트 작음) | 좋음 | 좋음 |

**최종 순위: D > C > A > B**

1. **D(추천)** — 유일하게 실제 앱 화면을 담아 "진짜 제품"이라는 신뢰를 즉시 전달하면서 슬로건+차별점(출발알림)까지 동시 전달. Play 리스팅에서 Feature Graphic이 스크린샷보다 먼저 노출되는 경우가 많아, 실체를 보여주는 것이 설치 결정에 유리.
2. **C** — 차별점 전달력은 D와 동급이나 추상 아이콘이라 신뢰도에서 밀림.
3. **A** — 무난하지만 스크린샷 1·8번과 메시지가 완전히 겹쳐 정보 가치가 낮음.
4. **B** — 정보량이 많아 축소 상태 가독성 리스크가 가장 큼.
