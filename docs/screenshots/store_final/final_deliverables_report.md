## 3. 스토어 사용 금지 과장 표현 목록

| 금지 표현 | 사유 | 실제 사용한 대안 |
|---|---|---|
| "~이 완성됩니다", "100%" | 인식 정확도 보장 함의 | "~이 됩니다" (정리 결과로 표현) |
| "모든 일정에 자동으로" | 실제로는 위치 좌표가 잡힌 일정에만 스마트 준비알람 동작 | "장소가 있는 일정은" (조건 명시) |
| "함께 확인" (동기화 완전성 함의) | 연동은 사용자 설정 필요, 동기화 시차 존재 | "연결하면 자동으로 동기화돼요" |
| "앱을 열 필요 없이" | 위젯도 실질적으로 앱 프로세스를 통해 동작 | "홈 화면에 바로 놓아요" |
| "최고", "1위", "베스트" 등 최상급 | Google Play 정책상 사용 제한 | 사용하지 않음 |
| "절대", "무조건", "보장" | 사실과 다를 시 표시광고법 저촉 위험 | 사용하지 않음 |

## 4. 디자인 선택 근거

- **1~3번(네이비) / 4~7번(라이트) / 8번(네이비) 리듬 구성**: 스토어 탐색 시 첫 3장에서 "가치 → 차별점 → AI 편의성 심화"로 몰입시키고, 이후 기능 투어는 밝은 톤으로 전환해 정보 열람 피로를 낮춤. 8번은 슬로건과 함께 브랜드 톤으로 마무리.
- **실기기 스크린샷 100% 사용**: 사용자 지시("가짜 UI 금지")에 따라 모든 화면은 S23 실기기 캡처 기반이며, 개인정보만 이미지 레벨에서 패치.
- **제목 위 여백**: 최초 210px → 105px(과도) → 최종 160px로 사용자 피드백 반영해 조정. 스크린샷 슬롯은 이 여백 감소분만큼 확대(1096px→1146px).
- **브랜드 컬러**: 앱 실제 테마(`lib/core/theme.dart`)의 `primary #1E3A5F`, `active #1A4FD6`, `background #EEF5FB` 그대로 사용.

## 5. 기존 스크린샷 대비 개선점

- 기존: 스토어 등록 스크린샷 없음(최초 제작).
- 이번 제작의 핵심 개선: (1) 모든 문구가 코드 검증을 거친 실제 동작 기반 (2) 조건부 표현으로 과장 리스크 제거 (3) 실기기 캡처로 UI 신뢰도 확보 (4) 8장이 하나의 서사(가치→차별→편의성→기능투어→브랜드)로 연결.
